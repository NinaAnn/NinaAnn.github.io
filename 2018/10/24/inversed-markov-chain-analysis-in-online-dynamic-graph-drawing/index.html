Inversed Markov Chain Analysis

motivation

Given a graph G(V,E), there exists many approaches to evaluate the importance of a node to the whole structure, for example in paper [1], the importance of a node $v$ is related to the degree and the edges’ weight which is connected to $v$. However, the importance of a node can’t only be estimated by its neighoring nodes and edges, but the infection it has to the whole structure.

As to the graph layout, the importance of a node can be simply considered as the influence to other nodes when it is moved. As shown in figure below, the whole layout depends on A more than D because when A moves, nodes B, C, G has a high possibility of moving. However, the iterative move is difficult to be evaluated since it may have a high time complexity.

Inverse markov chain

Since it is difficult in analysing the influence between nodes which are not neighbors of each other, a heuristic idea inspired from [2] based on the possibility of a node’s influence to it’s neighbors is introduced here.

Take the figure above as an example, A is connected to three other nodes, so each neighbor’s influence to A is $\frac{1}{3}$, which doesn’t exactly correspond to the reality because each neighbor’s move will cause A to move, however, A is also constraint to the two other nodes, so in order to simplify the model, we use the influence possibility.

edge ${uv}’s$ influence to a node $v$ is :
$$
I_{uv} =\frac{w_{uv}}{\sum_{i\in V_v}w_{iv}}
$$
with $w_{uv}$ edge $uv$’s weight, and $V_v$ ensemble of $v’s$ neighbors.

Hence the influence matrix $I$ is defined, indicating the influence of node $i$ to node $j$.

The influence matrix of figure above is :
$$
I =
\left [\begin{matrix}
0 & 1 & \frac{1}{2} & \frac{1}{3} & 0 & 0 & 0\\
\frac{1}{3} & 0 & 0 & 0 & 0 & 0 & 0\\
\frac{1}{3} & 0 & 0 & 0 & 0 & 0 & 1\\
\frac{1}{3} & 0 & 0 & 0 & \frac{1}{2} & \frac{1}{2} & 0\\
0 & 0 & 0 & \frac{1}{3} & 0 & \frac{1}{2} & 0\\
0 & 0 & 0 & \frac{1}{3} & \frac{1}{2} & 0 & 0\\
0 & 0 & \frac{1}{2} & 0 & 0 & 0 & 0
\end{matrix}\right ]
$$

It is obvious that the sum of each column is 1, moreover, we suggest that the movement of each node is only related to the status of the previous time step, so the movement fits markov condition, and the influence matrix $P$ is the transpose of a markov matrix.

convergence

Same as the markov chain, $P$ converges to a static matrix $\hat{P}$ as $P=transpose(M)$, and the value of each line is the same, and the sum of each column is 1.

According to the property of markov chain, we know that the $vM=v$ and eigen vector $v$ whose sum is 1 equals to each row’s value, so there exists
$$Iv^T=v^T$$
$$\sum{v^T}=1$$

We can easily find that one of the eigen value $1$’s eigen vector is
$$
v^T=\left[D_1,…,D_n \right]^T
$$
with $D_i$ the weighted degree of node $i$:
$$
D_i = \sum_{k\in V_i}w_{ik}
$$

Because each line $i$ of $Pv^T$ is :
$$
\sum_{j\in V}I_{ij}D_j=\sum_{j\in V}w_{ij}=D_i
$$

So the convergence of $P$ is:
$$
\left [\begin{matrix}
\frac{D_1}{\sum D_i} & \cdots & \frac{D_1}{\sum D_i}\\
\vdots & \cdots & \vdots\\
\frac{D_n}{\sum D_i} & \cdots & \frac{D_n}{\sum D_i}
\end{matrix}\right ]
$$

Node Importance

Node importance is evaluated here as the influence to the whole layout when the node is moved, so the probability of this node to move is 1 and the movement vector is:

$$p_0=[\begin{matrix}0,…,1,…,0\end{matrix}]$$

according to the inverse markov chain, the possibility that other nodes are influenced by this move is:

$$p_0I$$

In the $n^{th}$ step the possibility will be changed to
$$
p_n = p_0I^n
$$

When $n$ is big enough, $p_mI^n$ will converge to what we calculated before, which means that the whole layout reaches a pseudo-stability. The sum of $p_n$ shows the possibility of all nodes that will be influenced during the whole process.

However, the calculation of convergence takes too much time, moreover, we can consider the whole system as a string model without any energy loss, which will never reach a real stability when a force is added. So we append the idea of simulate annealing to simulate the energy loss:

$$
p_n = 0.8p_{n-1}I
$$

The whole system will quickly reach zero. By calculating the influence of each node in each step, we can simulate the influence of movement of node $i$ to all other nodes.

In graph layout analysis, we hope that the one with higher influence moves with the lower time, because it’s movement will cause a series of changes.

Finding connected components during analysis

Detecting connected components can use DFS(Deep First Search), Tarjan algorithms, however, by using node importance analysis by using inverse markov chain, we can find all connected components at the same time.

At each time we evaluate one node $i$, and we use uniform function $U$ to the total moving possibility:

$$
U(x)=\begin{cases}
0 &\text{$x=0$}\\
1 &\text{$x\gt 0$}
\end{cases}
$$

The result matrix is the group connected to $i$, in each evaluation, we update or merge the groups matrix using or(|) operation or create a new group, the whole process can be explained in the following algorithm:

As we can see, in each analysis of node, what to do to find connected components is just do a traversal to all the connected components that already found.

Calculation of time complex:

We suppose the maximum connected components found during the process is k. If we do node importance analysis on CPU, and the maximum iteration time is u, the total time complex is: $O(ukN^3)$. However, as we know, matrix multiplexion can be done in parallel algorithms, which can highly reduce calculation time, if we open N blocks and each block has N threads in GPU, the time complex can be reduced to $O(ukN)$.

Dynamic layout analysis

Instead of using inverse markov chain to analyse the node importance in a statistic graph, it can also be used to simulate the nodes’ movement when the layout changes.

When a new node is added, we think it as absolutly to move. When an edge is deleted, the reduce of an edge will not directly cause the layout’s change, however, when a new edge $ij$ is added, the possibility that the nodes it is connected to will move is the possibility that it will be infected by the new added edge.

$$
p_{i,move}=\begin{cases}
1 &\text{$i\in V_n/V_{union}$}\\
\sum_j\frac{w_{ij}}{D^{‘}_i} &\text{$ij\in E_n/E_{union}$}\\
0 &\text{$ij\in E_{n-1}/E_{union}$}\\
0 &\text{otherwise}
\end{cases}
$$

With $D_i$ degree of $i$ in graph $G_n\cap G_{n-1}$ and $D_i^{‘}$ is the degree in graph $G_n$

We do the same process to the new influence matrix $I’$ using the changing possibility decided before to simulate the whole change and find the nodes sensible to graph change and the ones which are insensible.

Optimization

Since what we want to do is to simulate the graph’s layout’s change, so the position’s influence to the node’s changing possibility should be taked into consideration.

When an edge is added, if the two nodes are far away from each other, they are more possible to move, the same, when an edge is deleted, if the related nodes have other neighbors and are far away from them, they are more possible to move.

Hence we change the possibility to:

$$
p_i =
\begin{cases}
\sum_{j}\frac{w_{ij}}{D^{‘}_i}\times \frac{|dis(ij)-l1|}{l_1} & \text{$j\in neigh(i)\cup i\in V_{union}$} \\
1 + \sum_{j\in V_{union}}\frac{w_{ij}}{D^{‘}_i}\times \frac{|dis(ij)-l1|}{l_1} & \text{$j\in neigh(i)\cup i\notin V_{union}$}\\
\sum_{k\in neighbors(i)}\frac{w_{ik}}{D^{‘}_i}\times \frac{|dis(ij)-l1|}{l_1} & \text{$ij\in E_{n-1}/E_{union}$}\\
0 & \text{otherwise}
\end{cases}
$$

$l_1$ is the ideal distance of two nodes with graph distance 1.

p_{move}=\left[\begin{matrix}

sumP_{move}=\left[\begin{matrix}

Our optimization greatly make node $C,G,E,F$’s change much bigger than other nodes, as G and F are very far from each other.

Constraint stress energy

As mentioned by KAMADA and KAWAI[3], the graph layout can be transfered in minimizing stress function:

$$
E=\sum_{i=1}^{n-1}\sum_{j=i+1}^n\frac{1}{2}k_{ij}(\Vert x_i-x_j\Vert -l_{ij})^2
$$

In the graph layout with constraint[4][5], we always add constraint functions to stress function, there are many kinds of constraint functions that we can add, including reduce edge cross or maintain specific shapes[6], there are three main constraint functions that can be used in general dynamic graph layout to preserve mental map.

  1. $min\sum(\Vert x_i-x_j\Vert-\Vert x_i^*-x_j^*\Vert)^2$ with $x_i,x_j$ new positions of nodes $i$ and $j$ and $x_i^*,x_j^*$ old positions of them.

    Aimed to maintain the former structure between nodes.

  2. $min\sum(\Vert x_i-x_i^*\Vert^2)$ with $x_i$ new position of node $i$ and $x^*_i$ old position.

    Aimed to maintain the former position of nodes exist in both layout.

  3. $min\sum\Vert x_i-x_j-(x_i^*-x_j^*)\Vert^2$

    proposed firstly in order to generate directed graph layout in [5], then used in [6] to preserve both direction and distance information between two nodes.

Node selection based on importance

During updating node positions, in order to keep the former structure as much as possible, at the same time, prevent new layout from being in mess, we set the following principle in deciding the nodes’ movement:

  1. the ones with higher influence $P_{influ}$ to the graph $G_n\cap G_{n-1}$ change less.
  2. the ones with higher moving possibilities $P_{move}$ change more.

Hence for each node, we assign a moving parameter $S:$

For node $n:$
$$
\hat{S_n} = \frac{P_{influ}}{P_{move}+0.1}
$$

$$
S_n = \alpha\hat{S_n}
$$

$0.1$ is added to prevent $P_{move}$ from being to small.

$\alpha$ is a parameter: $\alpha=max\hat{S_n}$, hence all $S_n$ is smaller than 1.

Structure selection based on edge

Using edge connectivity

Each element $I_{ij}$ in influence matrix is the possibility that a node $i$’s influence to node $j$, whose value can also be considered as whether node $i$ and $j$ are closely to each other. So a set of close nodes can form a relatively stable structure.

The relationship matrix $R$ is defined as:

$$
R=I+I^T
$$

So the edge $ij$’s stability parameter $S_{ij}$ is:

$$
\hat{S_{ij}}=min(R^{ij}_{G_n\bigcap G_{n-1}},R^{ij}_{G_n})
$$

$$
S_{ij}=\beta \hat{S_{ij}}
$$

$\beta =max\hat{S_{ij}}$, hence all $S_n$ is smaller than 1.

Using node connectivity

There is another approach in analysing edge stabilities by using node importance analysis. In the previous node importance analysis, the node $j$ which changes the most in analysing node $i$ seems to be the one highly connected to $i$, hence edge $ij$ or path between $i$ and $j$ is strongly connected.

The relationship matrix $R$ here is defined as:

$$
R=I_{res}+I_{res}^T
$$

With $I_{res}$ the result matrix of importance analysis:

$$
I_{res}[i,:] = sumP_i
$$

So the stability parameter between nodes $i$,$j$ is:

$$
\hat{S_{ij}}=min(R^{ij}_{G_n\bigcap G_{n-1}},R^{ij}_{G_n})
$$

$$
S_{ij}=\beta \hat{S_{ij}}
$$

$\beta =max\hat{S_{ij}}$, hence all $S_n$ is smaller than 1.

Analysis

Advantages:

  1. Give a simple way in analysing the node importance and simulating the movement of nodes in dynamic systems.
  2. Change traditional for-loop into matrix calculation, which can be easily implemented in distributed calculation systems.

Disadvantages:

  1. The dynamic simulation is not suitable for graphs with multiple connected components.
  2. Our inversed markov chain analysis is based on brief events, in long-lasting events[12] or some cases where we think events which last long time should be more stable[9], The current layout shouldn’t be considered as only related to the layout of previous one step

Solutions to the disadvantages:

In case of multi-connected components:

Dynamic simulation is not suitable for graphs with multiple connected components is basicly because in graph layout, we have to prevent overlapping, however, dynamic simulation is based on the connection, so it can’t predict cases of overlapping.

Solution:

Spring model in force directed layout with repulsion can’t applied here, so in order to prevent overlapping, we firstly update the layout, then seperate different connected components as inversed markov analysis can easily detect different connected components.

Dwyer[13] introduced a non-overlap constraint use the minimal penetration depth(mpd) first presented in [14], calculating mpd is a fast process using only O(m+n), however it only fits convex, hence in[13], a convex hull of each cluster has to be computed, whose time complex is about O(nlogn+mlogm).

Here we don’t need such a accurate seperation, hence we use the simplest approach: to let one connected component $i$ to move towards the direction of their barycenters $\overrightarrow{ji}$ distance $d=max(<n_i,\frac{ji}{\Vert ji\Vert}>)$

If two different connected components are connected by an edge:

The direction constraint is not suitable, because if the added edges make the whole structure change inversely, as shown in the figure below:

If EF and FG keep the former direction, it will make the new layout in a mess, in this case we propose two solutions:

  1. Change direction constraint to distance constraint on the smaller connected components(<e,f,g> here).

  2. Do pre-transmission to the smaller connected components, which means rotate it onto a suitable direction so that direction constraint can be applied in this case.

    Our pre-transmission process is simple: find the rotate angle of each edge and take their average to do rotation.

    • find the barycenter $c_i$ of all influenced nodes in all connected components $i$
    • for each edge $n_in_j$, calculate the innerproduct $I$ and cross value $Cr$ of $\overrightarrow{n_ic_i}$ and $\overrightarrow{n_jc_j}$
    • Rotation angle is $B=arcCos(I)$ and direction is $B$ if $Cr\gt 0$ else $360-B$
    • Final rotation angle is average(B)

In case of aging conditions:

Dynamic simulation is based on the idea of markov chain that the current situation is only concerned with the previous one. However, in some cases we want to take the previous layout into consideration, like age[9].

Solution:

Adding self-loop to the inverse markov matrix $P_{i\rightarrow i}$, indicating the manual-set stability of node $i$. Taking paper[9] as example, in the layout at time t, the node stability are updated based on:

$$
P_{i\rightarrow i}(t)=\begin{cases}
(P_{i\rightarrow i}(t-1)+0.1)(1-S_i) &\text{$P_{i\rightarrow i}(t)\lt \gamma$}\\
\gamma &\text{$P_{i\rightarrow i}(t) \gt \gamma$}
\end{cases}
$$

$P_{i\rightarrow i}$ of all newly added node is $0$.

Constraint stress energy minimization

Here, in order to preserve mental map[8], we use the second and third constraint function to preserve position, direction and distance information in or between nodes.

Stress function using edge connectivity

$$
\begin{aligned}
E&=\sum_{i=1}^{n-1}\sum_{j=i+1}^nw_{ij}(\Vert x_i-x_j\Vert -l_{ij})^2\\
&+\alpha_{node}\sum_{i\in V_n\bigcap V_{n-1}}S_i\Vert x_i-x_i^{*}\Vert^2\\
&+\alpha_{edge}\sum_{ij\in E_n\bigcap E_{n-1}}S_{ij}(\Vert x_i-x_j - (x_i^{*}-x_j^{*})\Vert)^2
\end{aligned}
$$

Stress function using node connectivity

$$
\begin{aligned}
E&=\sum_{i=1}^{n-1}\sum_{j=i+1}^nw_{ij}(\Vert x_i-x_j\Vert -l_{ij})^2\\
&+\alpha_{node}\sum_{i\in V_n\bigcap V_{n-1}}S_i\Vert x_i-x_i^{*}\Vert^2\\
&+\alpha_{path}\sum_{i,j\in V_n\bigcap V_{n-1}}S_{ij}(\Vert x_i-x_j - (x_i^{*}-x_j^{*})\Vert)^2
\end{aligned}
$$

Minimization Solutions

As mentioned in stress majorization[5], $E$ can be minimized by multiple approaches, such as Newton-Raphson method, conjugate gradient(CG) or cholesky decomposition. Among three methods, Newton-Raphson method needs to calculate second derivation, which takes much time, hence the mostly widely used methods are CG and Cholesky.

Solution by using stress majorization

We use local minimum here, so the first zero-derivation value is what we want,Here we use the second constraint stress function:

$$
\frac{\partial E}{\partial x}=\sum_{i\in N}\frac{\partial E}{\partial x_i}=0
$$

Applying stress majorization based on[5][7]:

$$
\begin{aligned}
E&\leq \sum_{i,j\in V_n}(w_{ij}l_{ij}^2+w_{ij}\Vert x_i-x_j\Vert^2-2w_{ij}l_{ij}inv(\Vert z_i-z_j\Vert)(x_i-x_j)^T(z_i-z_j)\\
&+\alpha_{path}\sum_{i,j\in V_n\cap V_{n-1}}(S_{ij}\Vert x_i^*-x_j^*\Vert^2+S_{ij}\Vert x_i-x_j\Vert^2-2S_{ij}(x_i-x_j)(x_i^*-x_j^*))\\
&+\alpha_{node}\sum_{i\in V_n\bigcap V_{n-1}}S_i\Vert x_i-x_i^*\Vert^2
\end{aligned}
$$

Which can be changed into matrix form:

$$
\begin{aligned}
E&\leq C+Tr(X^TLX)+Tr(X^TL_zZ)\\
&+\alpha_{path}(C’+Tr(X^TL_SX)-2Tr(X^TJD))\\
&+\alpha_{node}Tr((X-X^*)^TS(X-X^*))
\end{aligned}
$$

In order to minimize E, we have to set the derivative to zero:

$$
(L+\alpha_{path}L_S+\alpha_{node}S)X=L_zZ+\alpha_{path}JD+\alpha_{node}SX^*
$$

Then we iteratively find the minimum value using:

$$
(L+\alpha_{node}L_S+\alpha_{path}S)X(t+1)=L_{X(t)}X(t)+\alpha_{node}JD
$$

Then using CG or Cholesky to solve the linear equation and fix the right part using new solution $X(t+1)$

Apart from all those optimization methods, here we introduce a new solution based on Stochastic Gradient Descent.

Stochastic Gradient Descendent Minimization

The basic idea of SGD minimization is that we randomly choose one node each time to move a little step towards where E falls until the stress tends to convergence.

$$
x_i\leftarrow x_i-\eta \frac{\partial E}{\partial x_i}
$$

Here we use the second constraint stress function:

$$
\begin{aligned}
\frac{\partial E}{\partial x_i}&=2\sum_{j\in V_n}w_{ij}(x_i-x_j)-w_{ij}l_{ij}\frac{x_i-x_j}{\Vert x_i-x_j\Vert}\\
&+2\alpha_{path}\sum_{j\in V_n\bigcap V_{n-1}} S_{ij}(x_i-x_j-(x_i^*-x_j^*))\\
&+2\alpha_{node}S_i(x_i-x_i^*)
\end{aligned}
$$

The result can be rewritten to the matrix form:
$$
\begin{aligned}
\frac{\partial E}{\partial x_i}&=2(L_w[i,:]X-L_{w,d}[i,:]X)\\
&+2\alpha_{path}(L_S^{V_n\bigcap V_{n-1}}[i,:]X-L_S^{V_n\bigcap V_{n-1}}[i,:]X^*)\\
&+2\alpha_{node}S_i(X[i]-X^*[i])
\end{aligned}
$$

With

$$
\begin{aligned}
L_{w}(ij)&=\begin{cases}
\sum_{k\in V_n/i}w_{ik}&\text{$i=j$}\\
-w_{ij}&\text{$i\neq j$}\\
0&\text{Otherwise}
\end{cases}\\
L_{w,d}(ij)&=\begin{cases}
\sum_{k\in V_n/i}\frac{w_{ik}l_{ik}}{\Vert x_i-x_j\Vert }&\text{$i=j$}\\
-\frac{w_{ij}l_{ij}}{\Vert x_i-x_j\Vert }&\text{$i\neq j$}\\
0&\text{Otherwise}
\end{cases}\\
L_S^E(ij)&=\begin{cases}
\sum_{k\in E/i}S_{ij}&\text{$i=j$}\\
-S_{ij}&\text{$i\neq j\;\&\;i,j\in E$}\\
0&\text{Otherwise}
\end{cases}
\end{aligned}
$$

Optimization

Taking the advantage of SGD that each time we only change the position of one node, we can add some robust constraint in choosing which node to change instead of random selection.

Gaussian distribution selection

As analyzed before, the moving possibilities of different nodes can be totally different, if we randomly select nodes to move, much redundant calculation will be done.

Hence we can make the selection of nodes a gaussian distribution based on moving possibility, so some nodes will be selected much fewer times than others.

$$
P_i=\frac{exp\left(-p_i\right)}{\sum_{i\in V_n}exp(-p_i)}
$$

Without redundant calculation, the whole system converge more quickly.

mini-batch SGD

In the traditional SGD, each time we only update one node’s position, if the node selected has no need to change, the system will be considered as reaching convergence, however, there may still be many nodes who are in need of update, the result achieved by SGD update will hence be unreliable.

In order to solve the problem, the traditional SGD is changed to mini-batch SGD, meaning that each time a set of nodes will be selected.

Strict restriction of nodes’ positions

This restriction can strictly give restriction to nodes:

  1. Let one or multiple node stable during the movement.

    Sometimes we want to set a node as the center and keep it stable, in SGD minimization, we can set it’s probability of being chosen to 0.

  2. Set restriction to movement directions

    In some cases we want to keep some nodes move towards one direction, for example in tree layout generated by DigCola[6], in this case, set each node’s movement:
    $$
    x_i\leftarrow x_i-\eta <\frac{\partial E}{\partial x_i},(1,0)>
    $$
    can strictly constraint the nodes’ movement along x-axis.

Comparison to CG

Advantages:

During each iteration, CG has to go through all the nodes, however, in SGD, each time we only need to calculate derivation of one node. Of course the iteration time will be much more than CG, and the accuracy may be not so good as CG, but it saves much time in large-scale graph.

Secondly, by using only constraint function and CG solution, we can only give constraint, however, can’t have a strict constraint to node position or structure.

Disadvantages:

SGD is more difficult in applying to distributed systems like GPU(but not impossible). However GPU-based CG is easily to be applied.
